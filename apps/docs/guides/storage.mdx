---
title: "Storage"
description: "Persist session metadata with Drizzle-backed storage"
---

Flamecast persists session metadata and agent templates through Drizzle-backed storage.

## Backends

| Backend | Package | Use case |
|---|---|---|
| PGLite | `@flamecast/storage-psql` | Embedded Postgres. No external database needed |
| PostgreSQL | `@flamecast/storage-psql` | Production. Full Postgres with connection pooling |

## Configuration

`Flamecast` requires an explicit `storage` instance. In Node environments, `createPsqlStorage()` defaults to embedded PGLite when no `url` is provided.

### PGLite (default Node path)

```typescript
import { Flamecast } from "@flamecast/sdk";
import { createPsqlStorage } from "@flamecast/storage-psql";

const flamecast = new Flamecast({
  storage: await createPsqlStorage(),
});
```

PGLite stores data in `<cwd>/.flamecast/pglite` by default. Override with the `FLAMECAST_PGLITE_DIR` environment variable or the `dataDir` option.

Before first startup, apply migrations explicitly:

```bash
flamecast db migrate
```

### PostgreSQL

```typescript
import { createPsqlStorage } from "@flamecast/storage-psql";

const storage = await createPsqlStorage({
  url: "postgres://localhost/flamecast",
});

const flamecast = new Flamecast({ storage });
```

## Options

| Option | Description |
|---|---|
| `url` | Postgres connection string |
| `dataDir` | PGLite data directory (default: `<cwd>/.flamecast/pglite`) |

The storage package uses Drizzle ORM. Check or apply runtime migrations with:

```bash
flamecast db status
flamecast db migrate
flamecast db studio
```

Generate new migration files during development with:

```bash
pnpm psql:generate
```
